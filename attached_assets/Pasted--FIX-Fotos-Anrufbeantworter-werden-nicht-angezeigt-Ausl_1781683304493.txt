# FIX — Fotos & Anrufbeantworter werden nicht angezeigt (Auslieferung, nicht Speicher)

**Wichtig: Der Object Storage funktioniert — die Dateien WURDEN gespeichert (DB-Zeilen existieren). Der Fehler liegt allein beim Ausliefern. Nur das Folgende anfassen.**

## Hauptbug — Wildcard-Key in Express 5
Route `GET /api/theke/datei/*key` in `artifacts/api-server/src/routes/theke.ts` (um Zeile 451). In Express 5 ist `req.params["key"]` bei einem Wildcard ein **Array** von Pfadsegmenten (z. B. `["theke","foto","abc.jpg"]`), kein String. Der Code behandelt es als String:
```
const urlPath = (req.params as Record<string,string>)["key"] ?? "";
if (!urlPath.startsWith("theke/")) ...
```
→ `startsWith` greift nicht / der Key stimmt nicht → Datei nie gefunden → kaputtes Bild + Audio „Fehler".

**Fix:** den Key robust als String zusammensetzen, Array wie String behandeln:
```
const raw = (req.params as Record<string, unknown>)["key"];
const urlPath = Array.isArray(raw) ? raw.join("/") : String(raw ?? "");
```
Danach wie gehabt `urlPath.startsWith("theke/")`-Prüfung und `getFileBytes(urlPath)`.
- Falls Express die Segmente URL-codiert liefert, vor dem Join je Segment `decodeURIComponent` anwenden.
- Sicherstellen, dass die ausgelieferte Antwort den korrekten `Content-Type` setzt (Bild bzw. Audio) — ist bereits vorhanden, nach dem Key-Fix prüfen.
