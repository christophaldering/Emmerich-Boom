AUFGABE
Das feste Footer-Datum „Zuletzt geändert: 16. Mai 2026" durch einen automatisch beim Build
gesetzten Zeitstempel (Datum + Uhrzeit) plus Versionsnummer ersetzen. Jeder Relaunch/Build
soll den Wert frisch schreiben. Zusätzlich denselben Build-Stempel klein im Admin-Bereich
anzeigen.

TEIL 1 — Vite: Build-Zeit + Version injizieren
DATEI: artifacts/emmerich/vite.config.ts
- Vor dem export default zwei Werte berechnen:
    const BUILD_TIME = new Date().toISOString();
    Versionsstring: per try/catch `git rev-parse --short HEAD` (node:child_process execSync)
      ermitteln; schlägt das fehl, einen Zeitstempel-basierten Code als Fallback verwenden
      (z. B. aus BUILD_TIME abgeleitetes „vJJJJ.MM.TT-HHMM" in Europe/Berlin via
      toLocaleString("sv-SE", { timeZone: "Europe/Berlin" })).
- In defineConfig ein "define"-Feld ergänzen:
    define: {
      __BUILD_TIME__: JSON.stringify(BUILD_TIME),
      __BUILD_VERSION__: JSON.stringify(version),
    }
- Sonst nichts an der Vite-Config ändern (Plugins/Resolve/Server unverändert).

TEIL 2 — TypeScript-Deklaration der globalen Konstanten
DATEI: artifacts/emmerich/src/vite-env.d.ts (falls vorhanden, sonst neu anlegen)
- Ergänzen:
    declare const __BUILD_TIME__: string;
    declare const __BUILD_VERSION__: string;

TEIL 3 — Build-Info-Modul
NEUE DATEI: artifacts/emmerich/src/buildInfo.ts

      timeZone: "Europe/Berlin", day:"2-digit", month:"2-digit", year:"numeric",
      hour:"2-digit", minute:"2-digit",
