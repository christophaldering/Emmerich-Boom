**Befund: Der dauerhafte Hausmeister ist noch NICHT im Code (nur die 5 Beispiele + Leer-Rahmen). Der „Hausmeister”, den man bisher sah, ist das Demo-/Login-Profil — und das ist für andere ausgeblendet. Jetzt eine echte, permanente Figur einbauen, die JEDER sieht (auch wenn man als Christoph eingeloggt ist).**

## 1. Permanente Figur als Frontend-Fixture

In `artifacts/emmerich/src/pages/beispielProfile.ts` einen neuen Export `HAUSMEISTER` anlegen — analog zu `LEER_RAHMEN`, aber als sichtbare Porträt-Figur. Neues Flag `istInventar: true` (statt `istBeispiel`), damit er **nicht** entsättigt wird und ein eigenes Bändchen bekommt:


Das Feld `istInventar?: boolean` zum `GalerieEntry`-Typ in `Galerie.tsx` ergänzen.

## 2. Immer anzeigen (unabhängig von der Beispiel-Schwelle)

In `Galerie.tsx`, `alleEntries`-`useMemo` (≈ Zeile 450): den Hausmeister **immer** ans Ende hängen — sowohl wenn Beispiele gezeigt werden als auch wenn nicht:

```ts
const alleEntries = useMemo<GalerieEntry[]>(() => {
  const showBeispiel = entries.length < BEISPIEL_SCHWELLE;
  const base = showBeispiel
    ? [...entries, ...(beamer ? BEISPIEL_PROFILE : [...BEISPIEL_PROFILE, LEER_RAHMEN])]
    : [...entries];
  return [...base, HAUSMEISTER];   // Hausmeister immer, ganz rechts
}, [entries, beamer]);
```

## 3. Darstellung

- Porträt: statisches Bild `public/theke/hausmeister.jpg`, falls vorhanden; sonst Initialen „HK” auf Amber (NICHT entsättigt — er ist Inventar, kein Beispiel).
- Bändchen: statt „Beispiel” ein Bändchen **„Inventar”** (oder „gehört zum Haus”). Im Antipp-Fokus klar als Figur erkennbar.
- Behandlung wie ein normales Porträt bei der Sichtbarkeit (Rahmen sichtbar, z-index korrekt — Porträt über der Rahmenfläche).

## 4. Niemals zählen / nie „anwesend”

- Aus Präsenz/„X gerade an der Theke”, Band, Feed, DB und allen Statistiken heraushalten (wie die Beispiele). `anwesend` für `istInventar` immer `false` (glüht nie).
- Antippen: zeigt seinen Gag-Steckbrief im selben Overlay wie die Beispiele (kein echter Gast).
